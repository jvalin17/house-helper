# Architecture: User Auth & Multi-User Support

> Generated by /architecture on 2026-04-29
> Mode: standard
> Based on: requirements/user-auth-multiuser.md

## Architecture Pattern

Modular monolith with auth middleware layer. Auth is a separate module (`backend/auth/`) that wraps the existing app. Existing routes, services, and repositories are unchanged — the middleware swaps the DB connection per request based on the authenticated user.

## Component Diagram

```
Browser → React (AuthGuard + Login/Signup) → JWT in header
    → FastAPI → Auth Middleware (validate JWT, resolve user DB)
        → DB Router (connect to user's SQLite file)
            → Existing Routes (unchanged)
                → User's data.db (isolated)

Shared: auth.db (users table — email, password_hash, name)
Per-user: ~/.house-helper/users/{id}/data.db (full schema)
```

## Decision Log

| # | Decision | Choice | Depends On | Rationale |
|---|----------|--------|-----------|-----------|
| 1 | Auth method | Email + password + JWT | — | Simplest, no external services |
| 2 | Data isolation | Separate SQLite per user | — | Zero query changes, complete isolation |
| 3 | Connection routing | Dependency injection via get_conn() | #2 | Testable, backward compatible |
| 4 | API key encryption | AES-256-GCM | — | Authenticated encryption, standard |
| 5 | JWT storage | Authorization header | #1 | Stateless, no cookie config needed |
| 6 | Password hashing | bcrypt cost 12 | #1 | Industry standard, OWASP recommended |

## Data Architecture

### Auth DB (shared)
```sql
-- ~/.house-helper/auth.db
CREATE TABLE users (
    id INTEGER PRIMARY KEY,
    email TEXT UNIQUE NOT NULL,
    password_hash TEXT NOT NULL,
    name TEXT NOT NULL,
    created_at TEXT DEFAULT (datetime('now')),
    updated_at TEXT DEFAULT (datetime('now'))
);
```

### User DB (per user)
```
~/.house-helper/users/{user_id}/data.db
```
Same schema as current `house-helper.db` — all existing migrations run on creation. No schema changes to existing tables.

## Security

- **Passwords:** bcrypt, cost factor 12
- **JWT:** HS256 signed, 24h expiry, secret from `JWT_SECRET` env var
- **API keys:** AES-256-GCM encrypted, key from `ENCRYPTION_KEY` env var
- **HTTPS:** Required in production (reverse proxy)
- **No rate limiting yet** — park for future


## API Endpoints (New)

```
POST /api/auth/signup   { email, password, name }  → { token, user }
POST /api/auth/login    { email, password }        → { token, user }
GET  /api/auth/me                                  → { id, email, name }
PUT  /api/auth/me       { name }                   → { updated user }
POST /api/auth/logout                              → { ok }
```

All existing `/api/*` endpoints unchanged but require valid JWT.

## Data Flow

### Signup
```
POST /api/auth/signup → validate → bcrypt hash → INSERT auth.db
  → mkdir users/{id}/ → connect_sync(user_db_path) → migrate
  → issue JWT → return { token, user }
```

### Authenticated Request
```
GET /api/jobs → Auth middleware:
  → Extract Bearer token → validate JWT → get user_id
  → resolve path: ~/.house-helper/users/{user_id}/data.db
  → connect_sync(path) → attach to request.state.db_conn
  → route handler uses request.state.db_conn
  → response → close connection
```

### Login
```
POST /api/auth/login → find user by email (auth.db)
  → bcrypt.verify(password, hash)
  → issue JWT { user_id, exp }
  → return { token, user }
```

## Implementation Order

1. `backend/auth/models.py` — User dataclass + JWT claims
2. `backend/auth/db.py` — auth.db with users table
3. `backend/auth/service.py` — signup, login, hash, verify
4. `backend/auth/encryption.py` — AES-256-GCM for API keys
5. `backend/auth/middleware.py` — JWT validation + per-user DB routing
6. `backend/auth/routes.py` — auth endpoints
7. `backend/main.py` — wire middleware + auth routes
8. `frontend/src/hooks/useAuth.ts` — token management
9. `frontend/src/pages/Login.tsx` + `Signup.tsx`
10. `frontend/src/components/AuthGuard.tsx`
11. `frontend/src/App.tsx` — wrap routes in AuthGuard
12. Migration script — move existing data to first user account
